# 성원전기 세금계산서 변환 기능 개발 계획

이 문서는 국세청에서 내려받은 매출/매입 세금계산서 Excel 파일을 성원전기 내부 양식인 `세금계산서(2026년)-성원전기(주)` 형태로 변환하는 기능의 개발 계획이다.
이 기능은 특정 회사 전용 요구사항이므로, 나중에 다른 회사용으로 포킹할 때 제거될 가능성이 높다. 공통 ERP 기능과 최대한 분리해서 구현한다.

## 1. 목표

- 국세청에서 다운로드한 매출 세금계산서 `.xls` 파일을 업로드한다.
- 국세청에서 다운로드한 매입 세금계산서 `.xls` 파일을 업로드한다.
- 업로드한 파일을 파싱해 성원전기 내부 세금계산서 양식으로 변환한다.
- 변환 결과를 사용자가 다운로드할 수 있게 한다.
- 변환 전후 건수와 합계금액을 검증해 누락/중복 가능성을 화면에 표시한다.
- 국세청 파일만으로 알 수 없는 사내 관리값은 기본값, 규칙, 사용자 보정 단계로 분리한다.

## 2. 샘플 파일 확인 결과

확인한 파일:

```text
세금계산서(2026년)-성원전기(주).xls
성원전기(주)-06월 매입(국세청 다운).xls
성원전기(주)-06월 매출(국세청 다운).xls
```

국세청 다운로드 파일은 구형 Excel 바이너리 `.xls` 형식이다.
단순 CSV나 HTML 테이블이 아니므로 서버에서 `.xls`를 읽을 수 있는 파서가 필요하다.

## 3. 자동 변환 가능 범위

국세청 파일에서 바로 변환 가능한 값:

- `일자`: 국세청 `작성일자`
- `분기`: `작성일자` 월 기준 계산
- `거래처`: 상대 거래처 `상호 + 대표자명`
- `내역`: `품목명`
- `공급가액`: 국세청 `공급가액`
- `세액`: 국세청 `세액`
- `합계`: 국세청 `합계금액`
- `종류`: 세금계산서 여부에 따라 `전자` 또는 `전자계산서`
- `공사번호`: 국세청 `비고`의 `공사번호:389920263376` 같은 값을 `3899-2026-3376` 형태로 변환

국세청 파일만으로 확정하기 어려운 값:

- `종목`
- `구분`: 입금/외상
- `입금일`
- `입금액`
- `미수금액`
- `비고`
- 매출의 계약 관련 열
- 매입에 국세청 파일 외 별도로 들어가는 반복/수기 항목

1차 구현에서는 확정 어려운 값에 다음 정책을 둔다.

- `종목`: 거래처/품목명 기반 규칙이 있으면 자동 입력, 없으면 빈 값 또는 `미분류`
- `구분`: `영수/청구 구분`이 `영수`이면 `입금`, 그 외는 `외상` 기본값
- `입금일`: `영수`이면 작성일자 또는 품목일자, 그 외는 빈 값
- `입금액`: `입금`이면 합계금액, 그 외는 빈 값
- `미수금액`: `외상`이면 합계금액, `입금`이면 0
- `비고`: 국세청 `비고`를 그대로 넣지 않고, 사내 규칙에 맞는 값만 매핑

주의:

샘플 회사 양식의 입금/외상 상태는 국세청 `영수/청구 구분`만으로 설명되지 않는다.
실제 입금 상태까지 맞추려면 별도 입금자료, 기존 회사 양식 병합, 또는 화면에서 사용자 보정이 필요하다.

## 4. 분리 전략

이 기능은 공통 ERP 기능이 아니라 성원전기 전용 변환기능으로 본다.

### Backend

회사 전용 모듈을 별도 폴더로 둔다.

```text
backend/src/company-features/sungwon-tax-invoice-converter/
```

예상 구성:


공통 코드 변경은 최소화한다.

- `app.module.ts`에 모듈 import 1줄

- 필요 시 권한 seed 반영

- 공통 employee/business 기능과 엮지 않는다.

### Frontend

회사 전용 화면을 별도 폴더로 둔다.

```text

```

공통 변경은 최소화한다.

- `frontend/src/config/menus.ts`에 메뉴 1개
- page는 기존 구조대로 `AppShell` 흐름 안에서 사용
- 파일 업로드/미리보기/다운로드 UI는 회사 전용 컴포넌트에만 둔다.

### Feature Flag

환경변수로 기능 노출 여부를 제어한다.


다른 회사용 포킹에서는 환경변수로 즉시 숨기고, 이후 전용 폴더와 메뉴만 제거할 수 있게 한다.

### POST /convert

업로드 파일과 사용자 보정값을 받아 회사 양식 파일을 생성한다.

요청:

```text
multipart/form-data
salesFile: 국세청 매출 .xls
purchaseFile: 국세청 매입 .xls
options: JSON 문자열
```

응답:

```text
application/vnd.openxmlformats-officedocument.spreadsheetml.sheet
```

다운로드 파일명:

```text
세금계산서(2026년)-성원전기(주).xlsx
```

`.xls`가 반드시 필요하면 2차로 `세금계산서(2026년)-성원전기(주).xls` 출력 가능성을 검증한다.
서버에서 Excel COM에 의존하지 않기 위해 1차 구현은 `.xlsx` 출력을 우선 검토한다.

## 7. Excel 처리 방식

### 입력 파싱

국세청 파일이 `.xls`이므로 Node 패키지 도입이 필요하다.

후보:

- `xlsx`: 구형 `.xls` 읽기와 기본 workbook 생성 가능
- `exceljs`: `.xlsx` 작성과 스타일 제어에 유리하지만 `.xls` 입력은 직접 읽지 못함

1차 권장안:

- 입력 `.xls` 파싱은 `xlsx` 사용
- 출력은 `.xlsx` 템플릿을 `exceljs`로 채워 스타일을 유지
- 기존 `.xls` 샘플 양식은 개발 전에 `.xlsx` 템플릿으로 한 번 변환해 `templates/`에 보관
- 실제 회사 데이터가 들어간 샘플 파일은 저장소에 커밋하지 않고, 구조만 반영한 빈 템플릿을 커밋

### 출력 생성

출력 파일은 다음 방식 중 하나로 만든다.

1. 템플릿 기반 생성
   - 빈 회사 양식 `.xlsx` 템플릿을 둔다.
   - `매출장`, `매입장`, `요약`, `계산서(매입)`, `계산` 시트를 유지한다.
   - 데이터 영역만 채우고 스타일을 유지한다.

2. 코드 기반 생성
   - 시트, 헤더, 스타일을 코드로 생성한다.
   - 템플릿 관리가 필요 없지만, 기존 양식과 100% 동일하게 맞추는 비용이 크다.

1차 구현은 템플릿 기반 생성을 우선한다.

## 8. 변환 매핑 정책

### 공통

- 날짜는 내부 값으로는 실제 날짜를 유지하고, 표시 형식은 `yy-mm-dd` 또는 샘플의 `'26-06-30` 형태를 맞춘다.
- 분기는 작성일자 기준으로 계산한다.
  - 1월~3월: `1분기`
  - 4월~6월: `2분기`
  - 7월~9월: `3분기`
  - 10월~12월: `4분기`
- 행 정렬은 작성일자 오름차순을 기본으로 한다.
- 금액은 문자열이 아니라 숫자로 저장한다.
- 변환 후 원본 국세청 합계와 생성 파일 합계를 비교한다.

### 매출

국세청 매출 행에서 회사 `매출장`으로 매핑한다.

```text
작성일자 -> 일자
상대 거래처 상호 + 대표자명 -> 거래처
품목명 -> 내역
공급가액 -> 공급가액
세액 -> 세액
합계금액 -> 합계
전자세금계산서분류 -> 종류
비고의 공사번호 -> 공사번호
```

공사번호 변환 예:

```text
공사번호:389920263376한전사업소명:경기본부/서수원지사
-> 3899-2026-3376
```

### 매입

국세청 매입 행에서 회사 `매입장`으로 매핑한다.

```text
작성일자 -> 일자
공급자 상호 + 대표자명 -> 거래처
품목명 -> 내역
공급가액 -> 공급가액
세액 -> 세액
합계금액 -> 합계
전자세금계산서분류 -> 종류
```

거래처명 표기 차이는 규칙으로 정리한다.

예:

- `(주)한솔이엠티` -> `㈜한솔이엠티`
- `주식회사 무진에너지` -> `㈜무진에너지`
- `임현정 외1명` -> `임현정 외 1 명`

이런 표기 변환은 1차부터 모두 완벽히 맞추기보다, 별도 `partner-name.rules.ts` 또는 화면 보정으로 점진 적용한다.

## 9. 사내 보정값 처리

국세청 파일에 없는 사내 관리값은 다음 우선순위로 처리한다.

1. 사용자가 화면 미리보기에서 직접 수정한 값
2. 회사 전용 거래처/품목 규칙
3. 기본값

규칙 예:

```text
거래처가 한국전력공사이고 비고에 공사번호가 있으면 종목 = 단가(2025)
품목명에 장비사용료가 있으면 매입 종목 = 건설화물 또는 건설기계 후보
거래처가 KT이면 종목 = 경비
거래처가 임대 관련이면 종목 = 경비
```

확정이 어려운 규칙은 자동 확정하지 않고 `미분류`로 표시한다.

수기/반복 매입 처리 후보:

1. 화면에서 추가 행 입력
2. 별도 `추가 매입` Excel 업로드
3. 전월 또는 기존 회사 양식에서 반복 항목을 가져오기

1차 구현에서는 화면에서 추가 행을 입력할 수 있게 하거나, 아예 제외하고 경고만 표시하는 방식 중 하나를 선택한다.

## 10. Frontend 화면 계획

화면명:

```text
세금계산서변환
```

구성:

1. `PageHeader`
2. 변환 기준 입력
   - 연도
   - 월
3. 파일 업로드 영역
   - 국세청 매출 파일
   - 국세청 매입 파일
4. 미리보기 요약
   - 매출 건수/공급가액/세액/합계
   - 매입 건수/공급가액/세액/합계
   - 원본 헤더 합계와 행 합계 일치 여부
5. 보정 필요 목록
   - 종목 미분류
   - 입금/외상 기본값 적용 행
   - 공사번호 추출 실패 행
6. 변환 실행 버튼
7. 다운로드 버튼

버튼:

- 파일 선택
- 미리보기
- 변환 파일 다운로드
- 선택 초기화

1차 UI 정책:

- 파일은 서버에 영구 저장하지 않는다.
- 미리보기 후 바로 다운로드 가능하게 한다.
- 미분류/외상 기본값이 있어도 다운로드는 허용하되 경고를 표시한다.

2차 UI 후보:

- 미리보기 표에서 `종목`, `구분`, `입금일`, `입금액`, `비고`를 직접 수정
- 수정값을 반영해 다운로드
- 자주 쓰는 거래처 규칙 저장

## 11. 데이터 저장 정책

1차 구현은 stateless 변환으로 시작한다.

- 업로드 파일을 DB에 저장하지 않는다.
- 생성 파일을 DB에 저장하지 않는다.
- 변환 요청 중 임시 파일이 필요하면 OS temp 또는 메모리에서 처리하고 즉시 삭제한다.
- 변환 이력 테이블은 만들지 않는다.

추후 이력이 필요하면 회사 전용 테이블을 추가한다.

```text
sungwon_tax_invoice_conversion_batches
sungwon_tax_invoice_conversion_files
sungwon_tax_invoice_partner_rules
```

이 테이블도 `company-features` 전용으로 관리하고, 공통 ERP 회계 테이블로 확장하지 않는다.

## 12. 보안/개인정보/파일 처리

- 국세청 파일에는 사업자번호, 주소, 이메일, 거래처 정보가 포함된다.
- 업로드 원본 파일명과 주요 합계 정도만 로그에 남기고, 행 단위 거래처/이메일/주소를 로그에 남기지 않는다.
- 파일 크기 제한을 둔다.
- 허용 확장자는 1차로 `.xls`, `.xlsx`만 둔다.
- MIME type만 믿지 않고 실제 workbook 파싱 성공 여부를 검증한다.
- 변환 실패 시 원본 행 전체를 에러 메시지로 반환하지 않는다.
- 권한이 있는 사용자만 접근하도록 기존 `JwtAuthGuard`, `MenuPermissionGuard`를 사용한다.

## 14. 미확정 사항

진행 전 확인이 필요한 항목:

1. 다운로드 파일 확장자가 반드시 `.xls`여야 하는지, `.xlsx`도 가능한지
2. 국세청 파일 외 수기/반복 매입 2건을 1차 구현에서 어떻게 넣을지
3. `종목` 자동 분류 규칙을 어디까지 1차에 넣을지
4. `입금/외상`, `입금일`, `입금액`, `미수금액`을 자동 기본값으로 둘지, 화면에서 수정 후 다운로드하게 할지
5. 기존 회사 양식 파일을 업로드해서 그 안의 수기값/반복항목/스타일을 이어받는 기능이 필요한지
6. `계산서(매입)` 시트에 들어갈 국세청 계산서/현금영수증 자료도 이번 범위에 포함할지

## 16. 1차 구현 결정안

- 회사 전용 기능으로 `sungwon-tax-invoice-converter` 폴더에 격리한다.
- 업로드 파일은 DB에 저장하지 않는 stateless 변환으로 시작한다.
- 국세청 `.xls` 매출/매입 파일을 파싱한다.
- 출력은 우선 `.xlsx`로 생성하고, `.xls` 필수 여부는 구현 전 확인한다.
- 매출/매입 원본 건수와 합계를 화면에서 검증한다.
- 국세청 파일만으로 확정 불가능한 값은 기본값과 경고로 처리한다.
- 매입 수기/반복 항목은 1차에서 제외하거나 화면 추가 입력으로 처리한다.
- 실제 세금계산서 샘플 파일은 민감한 거래처 정보가 있으므로 저장소에 커밋하지 않는다.
